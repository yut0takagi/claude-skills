勉強会の企画・準備・振り返りを支援してください。

引数: $ARGUMENTS
（例: `/勉強会運営 Claude Code入門 3回シリーズの企画` `/勉強会運営 次回の資料準備` `/勉強会運営 前回の振り返り`）

---

## フェーズ判定

| フェーズ | 判定条件 | 処理 |
|---------|---------|------|
| **企画** | 「企画」「新しい」「シリーズ」「カリキュラム」 | ステップ1へ |
| **準備** | 「準備」「資料」「次回」「スライド」 | ステップ3へ |
| **振り返り** | 「振り返り」「実施後」「フィードバック」 | ステップ5へ |

---

## ステップ0：前提情報

- `${VAULT_PATH}/自分について.md` — プロフィール・所属
- `${VAULT_PATH}/議事録/` — 過去の勉強会関連議事録を検索
- `${VAULT_PATH}/ナレッジ/README.md` — 活用できるナレッジ

---

# 企画フェーズ

## ステップ1：カリキュラム設計

ユーザーのテーマと以下を確認：
- **対象者:** 誰向けか（新卒/中途/非エンジニア等）
- **回数・時間:** 何回シリーズか、1回あたり何分か
- **ゴール:** 参加者が最終的に何ができるようになるか

### 設計テンプレート

```
## 勉強会カリキュラム: {テーマ}

**対象:** {対象者}
**回数:** 全{N}回（各{M}分）
**ゴール:** {到達目標}

### 各回の内容

| 回 | テーマ | 内容 | 成果物 |
|----|--------|------|--------|
| 1 | {テーマ} | {内容} | {ハンズオン成果物} |
| 2 | ... | ... | ... |
```

## ステップ2：Slack告知文の生成

```
【勉強会のご案内】

{テーマ} 勉強会を開催します！

📅 日時: {日時}
📍 場所: {場所/オンライン}
👥 対象: {対象者}
🎯 ゴール: {ゴール}

全{N}回シリーズ:
{各回の1行サマリー}

参加希望の方はスレッドにリアクションお願いします！
```

# 準備フェーズ

## ステップ3：資料準備

過去の勉強会議事録・ナレッジから関連情報を収集し、以下を準備：

1. **スライド:** `/スライド作成` と連携してスライドを生成
2. **ハンズオン手順:** ステップバイステップの手順書を作成
3. **デモ環境:** 必要なセットアップ手順を整理

## ステップ4：事前チェックリスト

```
### 勉強会準備チェックリスト

- [ ] スライド完成
- [ ] ハンズオン手順の動作確認
- [ ] デモ環境のセットアップ確認
- [ ] 会議室/Zoom予約
- [ ] Slack告知済み
- [ ] 参加者人数確認
```

---

# 振り返りフェーズ

## ステップ5：振り返りナレッジ化

勉強会実施後、以下を整理：

1. **参加者数・フィードバック** を収集
2. **うまくいったこと / 改善点** を整理
3. **次回に向けた改善案** を提案
4. ナレッジファイルとして保存（`/ナレッジ整理` と連携）

---

## ステップ6：完了報告

フェーズに応じた成果物を報告。

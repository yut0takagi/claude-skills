全プロジェクトの進捗を一覧化してください。

引数（任意）: $ARGUMENTS
（例: `/プロジェクト進捗` `/プロジェクト進捗 商品A`）

---

## ステップ0：前提情報を読み込む（並列実行）

まず `${VAULT_PATH}/自分について.md` を読み込み、プロジェクト一覧を把握した上で、以下の **3つのサブエージェント** を **1つのメッセージで同時に起動** してデータ収集してください（subagent_type: "general-purpose"）：

**Agent A — 日報収集：**
```
${VAULT_PATH}/日報/ から直近の日報3件を読み、以下を返してください：
- 各日報のプロジェクト別の活動内容
- 短期・中期目標
- 未完了のTODO
```

**Agent B — 週次レビュー収集：**
```
${VAULT_PATH}/日報/週次レビュー-*.md から最新1件を読み、以下を返してください：
- 各プロジェクトの進捗サマリー
- 来週やることリスト
- 目標達成度
```

**Agent C — 議事録収集：**
```
${VAULT_PATH}/議事録/ から直近2週間分の議事録をすべて読み、プロジェクト別に以下を返してください：
- 各プロジェクトの関連議事録タイトル・日付
- 主な決定事項・アクションアイテム
- 関係者
- 期限のあるタスク
```

### 結果の統合

全サブエージェントの完了後、メインエージェントが結果を統合してステップ1のプロジェクト集約に進む。

---

## ステップ1：プロジェクト単位で情報を集約

`自分について.md` の「主な活動・関わっているプロジェクト」に記載された各プロジェクトについて、以下を収集してください：

1. **現在のステータス**（日報・議事録から推定）
2. **直近のアクション**（最後に言及された日報/議事録の日付と内容）
3. **次のアクション**（未完了のタスク・TODOから抽出）
4. **期限**（明示されているもの、または推定）
5. **関係者**（誰とやっているか）

`$ARGUMENTS` で特定のプロジェクト名が指定されている場合は、そのプロジェクトだけ深掘りしてください。

---

## ステップ2：進捗レポートを出力

以下のフォーマットで出力してください：

```
## プロジェクト進捗レポート（{日付}時点）

### 全体サマリー

| プロジェクト | ステータス | 最終更新 | 次のアクション | 期限 | 優先度 |
|-------------|-----------|---------|--------------|------|--------|
| {名前} | {進行中/停滞/完了待ち} | {最終更新日} | {次にやること} | {あれば} | {高/中/低} |
| ... | ... | ... | ... | ... |

---

### {プロジェクト名1}

**ステータス:** {進行中/停滞/完了待ち/未着手}
**最終更新:** {最後に言及された日付}（{出典}）
**関係者:** {名前}

**直近の動き:**
- {箇条書き}

**次のアクション:**
- [ ] {内容}（期限: {あれば}）

---

### {プロジェクト名2}
...

---

## 注意事項・リスク
- {**最終更新から7日以上経過**しているプロジェクトは「停滞注意」と警告表示}
- {**最終更新から14日以上経過**しているプロジェクトは「要確認」と警告表示}
- {期限が近いタスクがあればハイライト}

---

## おすすめアクション
- {優先度が高い順に、今日・今週やるべきことを2〜3件提案}
```

---

## ステップ3：フォローアップ

出力後、以下を確認：
- 「プロジェクトの追加・完了・優先度変更はありますか？」
- 変更があれば `/自分更新` の実行を提案

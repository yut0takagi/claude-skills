提案資料のストーリー構成を自動検証し、改善ポイントを指摘してください。

引数: $ARGUMENTS
（例: `/提案構成チェック スライド/提案/クライアント提案.pptx` `/提案構成チェック 今日作ったスライドの構成をチェックして`）

---

## 検証フレームワーク

ScalpD提案の6回レビューで繰り返し指摘されたパターンから構築した検証軸：

### 推奨ストーリー構成（3/26議事録より）

```
1. 前提・与件整理（クライアントの状況・目的）
2. 課題の特定（データに基づく問題提起）
3. 要因分析（なぜその課題が起きているか）
4. 解決策（課題に直結する施策）
5. 実行計画（シミュレーション・予算・スケジュール）
6. 期待効果（KPI・次のステップ）
```

## ステップ2：4軸で自動検証

**Agent ツール** で4人のチェッカーを **並列起動**：

### チェッカー1: ストーリーフロー
- 「前提→課題→データ→解決策」の流れになっているか
- セクション間に論理飛躍がないか
- 「なぜ？」で繋がるか（A→B→Cの因果関係）

### チェッカー2: 1スライド1メッセージ
- 各スライドに明確なメッセージが1つあるか
- 複数メッセージが混在していないか
- ヘッダーがメッセージとして機能しているか（「分析結果」ではなく「若年層の購買率は40代の半分」）

### チェッカー3: データ・根拠の紐付き
- 主張に対するデータがあるか
- データの出典・N数・期間が明記されているか
- 「So What?」（だから何？）が明確か

### チェッカー4: 全体像→詳細の構成
- 全体像（サマリー/概要）が詳細の前にあるか
- 「Who × What」の戦略概要スライドがあるか
- 詳細データは本編かアペンディクスか適切に配置されているか
